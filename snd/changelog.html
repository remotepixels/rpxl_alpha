  • Added audio cues for when peers join or leave
  • Increased room entropy to 24 random alpha-numeric, upper and lower case, characters - 6224
  • Increased file ID entroby to 8 random alpha-numeric, upper and lower case, characters
  • Stopped peers from broadcasting their file lists if they don't have any
  • Optimized broadcasting of file lists, peers now only send their files not the entire list to each other

  • Cleaned up some log messages to make some things clearer
  • Handle local folders with same names differently, they are no longer merged together
  • Dragging the same folder in mutiple times will now create a separate folder each time instead of merging contents

  • Fixed issue where if remote added folder with same name they could remove it and it would remove link from all peers
  • Fixed an issue where empty folders were not being removed remotely
  • Missing files are now marked as dead on all peers if the caching fails on the host
  • Changed the file transfers to use raw binary instead of base 64 for 33% smaller file transfers
  • Saving files chunks via indexDB for temporary storage
  • Cached chunks are kept for 2 minutes after last host finishes transfers
  • Using indexDB to store files instead of in memory, this will allow much larger file transfers
  • Simple CRC for each chunk and resend of individual damaged chunks
  • Recieved chunks also stored in indexDB to lessen memory use with large files
